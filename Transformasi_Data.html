Digunakan untuk membuat basis data sederhana yang berisi informasi tentang bunga Iris, salah satu dataset populer di bidang machine learning.

  • Pembuatan database

Dibuat sebuah database bernama petal_mysql.

Database berisi tabel petalmysql dengan kolom:

  1. id â†’ primary key, bertipe integer dengan auto-increment.

Informasi ini bisa dipakai untuk analisis, misalnya membandingkan karakteristik antar spesies atau menguji algoritma klasifikasi.

  • Pembuatan database

Dibuat database bernama sepal_postgresql beserta tabel sepal_postgre.

  • Struktur tabel

Tabel memiliki empat kolom utama:

  1. id â†’ primary key, bertipe integer, memastikan tiap baris unik.

  2. class â†’ varchar, dipakai untuk menyimpan spesies bunga.

  3. sepal_length â†’ float, menyimpan panjang sepal.

  4. sepal_width â†’ float, menyimpan lebar sepal.

  • Fungsi struktur

  1. id menjaga keunikan data.

  2. class menunjukkan kategori spesies.

  3. sepal_length dan sepal_width mewakili variabel numerik yang dianalisis.

  • Pengisian data

Tabel diisi menggunakan perintah INSERT INTO dengan total 150 baris dari dataset Iris.

  • Isi data

Data mencakup tiga spesies bunga (Iris-setosa, Iris-versicolor, Iris-virginica), masing-masing berjumlah 50 sampel.

  • Format baris

Setiap baris berisi kombinasi spesies bunga dengan nilai panjang dan lebar sepal yang bervariasi.

  • Kegunaan

Struktur ini umum digunakan dalam analisis data dan machine learning karena menggabungkan data kategorikal (spesies) dengan data numerik (pengukuran).

  • Tujuan utama kode

Mengambil data dari dua basis data berbeda (PostgreSQL dan MySQL) menggunakan Python serta library pandas.

  • Bagian PostgreSQL

  1. Koneksi dibuat menggunakan library psycopg2.

  2. Query dijalankan untuk mengambil kolom: id, class, sepal_length, dan sepal_width dari tabel sepal_postgree.

  3. Hasil query dimasukkan ke DataFrame pandas (df_sepal) supaya mudah dianalisis.

  4. Setelah selesai, koneksi ditutup agar tidak memboroskan sumber daya.

  • Bagian MySQL

  1. Koneksi dilakukan menggunakan library pymysql.

  2. Query mengambil kolom: id, petal_length, dan petal_width dari tabel petal pada database petal_mysql.

  3. Hasil query disimpan dalam DataFrame df_petal untuk memudahkan pengolahan data.

  4. Setelah data didapat, koneksi ke MySQL juga ditutup.

  • Manfaat pendekatan ini

Data dari PostgreSQL dan MySQL bisa digabungkan atau dianalisis terpisah di Python, sehingga mendukung integrasi informasi dari berbagai sumber.

import pandas as pd
from sqlalchemy import create_engine

# PostgreSQL
engine_pg = create_engine("postgresql+psycopg2://postgres:amelia30@localhost/postgres")
df_sepal = pd.read_sql("SELECT id, class, sepal_length, sepal_width FROM sepal_postgree", engine_pg)

# MySQL
engine_mysql = create_engine("mysql+pymysql://root:@localhost/petal_mysql")
df_petal = pd.read_sql("SELECT id, petal_length, petal_width FROM petalmysql", engine_mysql)

df_merged = pd.merge(df_sepal, df_petal, on='id', how='inner')

for col in ['sepal_length', 'sepal_width', 'petal_length', 'petal_width']:
    df_merged[col] = df_merged[col].astype(float)

  • Tujuan kode

Mengambil serta menggabungkan data dari dua database berbeda (PostgreSQL dan MySQL) menggunakan Python dengan bantuan pandas dan SQLAlchemy.

  • Proses PostgreSQL

  1. Membuat engine koneksi memakai create_engine.

  2. Menjalankan query untuk mengambil kolom: id, class, sepal_length, sepal_width dari tabel sepal_postgree.

  3. Menyimpan hasil query ke dalam DataFrame df_sepal.

  • Proses MySQL

  1. Membuat engine koneksi ke database petalmysql.

  2. Mengambil kolom: id, petal_length, petal_width dari tabel petalmysql.

  3. Menyimpan hasilnya ke DataFrame df_petal.

  • Keunggulan pendekatan ini

Memungkinkan akses data dari berbagai sumber dengan format seragam DataFrame di Python.

  • Penggabungan data

  1. Data dari df_sepal dan df_petal digabungkan memakai inner join berdasarkan kolom id.

  2. Hasilnya berupa DataFrame df_merged, yang hanya menyimpan baris dengan id cocok di kedua tabel.

  • Penyiapan untuk analisis

  1. Kolom numerik (sepal_length, sepal_width, petal_length, petal_width) dikonversi ke tipe float.

  2. Data yang sudah bersih dan konsisten siap dipakai untuk analisis maupun perhitungan lebih lanjut.
